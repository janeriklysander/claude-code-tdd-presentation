(So I Don't Have To)

Jan-Erik Lysander

The "Why": My Experiment's Goals

  • ➤ As engineers, we strive for deep work, but context-switching is a constant battle between Linear, IDEs, GitHub, etc.
  • ➤ This back-and-forth creates friction and mental overhead.
  • Primary Goal: Could an AI assistant like Claude reduce this friction and help me stay in a state of flow?
  • Secondary Goal: Could it help enforce best practices, like TDD, that require active self-discipline?

Experiment Pt. 1: The First Test

Task: Create a script to parse, group, and sort messy Grafana JSON logs.

Why? To provide clean, relevant logs for a partner's OAuth flow investigation.

~1 hour

Manual Estimate

~15 mins

Actual Time with Claude

Key Finding: It wasn't a "one-shot" solution. It was a true collaborator, requiring iteration.

Disclaimer: No Pleo data was harmed (read: used) during this experiment.

Experiment Pt. 2: The "Aha!" Moment

Task: Build a browser extension to replace the deprecated GitHub Command Palette.

I tried to enforce a strict Red/Green TDD cycle, and then this happened:

  1. I acted as the reviewer.
  2. Claude wrote the failing test.
  3. Claude wrote the implementation to make it pass.
  4. Then, using gh CLI, it created a branch, committed the changes, pushed, and opened the Pull Request.

An incredible "flow state" where I directed and the AI executed the entire workflow.

The Good Bits

  • Unprecedented Speed: TDD cycles were incredibly fast.
  • Reduced Cognitive Load: My role shifted from "writer" to "reviewer & strategist."
  • Seamless IDE Integration: VSCode extension allows for live editing of proposed changes.
  • Automated Chores: It handled the entire Git flow with ease.

The Nuances & Friction

  • Context is King: It can stick to TDD, but requires frequent context clearing.
  • Sub-optimal TDD Cycle: It defaults to writing many tests at once, then making them all pass.
  • The Goal: The ideal is a one-test-at-a-time cycle, which needs better prompting.

Key Takeaways & Next Steps

Key Takeaways

  • Reduces Cognitive Load: Frees up mental space for higher-level problem-solving.
  • It's a Collaborator: The best results come from partnership, not delegation.
  • Process is Key: We need to develop new ways of working and standardized prompts.

Proposed Next Steps for Pleo

  1. Onboard once procurement is finalized.
  2. Develop shared CLAUDE.md best practices for a one-test-at-a-time TDD cycle.
  3. Leverage the VSCode extension for a more integrated review workflow.
