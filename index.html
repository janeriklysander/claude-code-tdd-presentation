Experiment Pt. 1: The First Test

Task: Create a script to parse, group, and sort messy Grafana JSON logs.

Why? To provide clean, relevant logs for a partner's OAuth flow investigation.

The Learning Curve: Chaos vs. Collaboration

Attempt 1: Chaos (Auto-Pilot)

Input: A large, vague Product Requirements Document.

Process: "Auto-accept" mode. No guidance, no TDD.

Output: Confused, duplicated code (ESM vs CJS), obsolete files, and frustration.

Attempt 2: Collaboration (Co-Pilot)

Input: Small, well-defined GitHub issues.

Process: Human as reviewer. Clear context. TDD enforced via CLAUDE.md.

Output: Modular code, better separation of concerns, and a predictable, successful workflow.

Treat the AI like a junior engineer: Double down on fundamentals. Break down work, be clear, and provide context.

The "Aha!" Moment

Task: Build a browser extension to replace the deprecated GitHub Command Palette.

With a disciplined approach, this became possible:

  1. I acted as the reviewer.
  2. Claude wrote the failing test.
  3. Claude wrote the implementation to make it pass.
  4. Then, using gh CLI, it created a branch, committed the changes, pushed, and opened the Pull Request.

An incredible "flow state" where I directed and the AI executed the entire workflow.

The Good Bits

  • Unprecedented Speed: TDD cycles were incredibly fast.
  • Reduced Cognitive Load: My role shifted from "writer" to "reviewer & strategist."
  • Seamless IDE Integration: VSCode extension allows for live editing of proposed changes.
  • Automated Chores: It handled the entire Git flow with ease.

The Nuances & Friction

  • Context is King: It can stick to TDD, but requires frequent context clearing.
  • Sub-optimal TDD Cycle: It defaults to writing many tests at once, then making them all pass simultaneously.
  • The Goal: The ideal is a one-test-at-a-time cycle, which needs better prompting.

How Can You Get Started?

1. Start Small

Pick one repetitive coding task you do daily. Experiment with Claude to reduce that toil and learn its patterns.

2. Be an Active Partner

Don't just delegate. Guide the AI, review its work, and step in when it goes off track. You are the senior engineer in the pair.

3. Document Your Process

Create your own instruction patterns (like in CLAUDE.md) and workflows. Share what works with your teammates.
